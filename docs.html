How the .md works

The builder turns one Markdown file into a finished, Outlook-ready newsletter. You (or Claude) fill in the file; the builder does all the layout.

1. Start from the template

Download the content template and fill it in — the fastest way is to hand it to Claude along with your notes, links, and event details, and ask it to fill in the blanks. The template explains itself as you go.

2. The format, in four rules

Every line of the file is one of these:

# Featured Research ← # is a SECTION (don’t rename these) ## Research Brief ← ## is a GROUP inside the section ### Brief #1 ← ### starts one ITEM (the label text is ignored) **Title:** Can Tutoring Move the Needle? **Authors:** Jane Doe & Sam Smith **Summary:** One or two sentences about the finding. **Url:** [Read the brief](https://example.com/brief)

Fields are matched by their label — Title, Authors, Summary, Date, Time, Location, Source, Url, Metaline — so keep the labels and change the values. Delete any field you don’t need, delete whole items you don’t have, and delete an entire # section if it’s empty this issue.

Two parts are required every issue: the loose date: line under the top title, and the # Intro: block (2–3 welcoming sentences).

3. From file to sent newsletter

  1. Upload — drop your finished .md into the builder.
  2. Outline — set the order of items, and pick one Featured event.
  3. Preview & Edit — click any text in the preview to fix it in place.
  4. Export — click Copy HTML. Then in Outlook Web App, start a new message, choose Insert → HTML, and paste. Send yourself a test first — it should look identical.
